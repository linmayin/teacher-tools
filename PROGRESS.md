### 開發歷程

#### v1.0｜初版建立
- 五個班級架構，班級名稱可修改
- 成績項目設定，支援跨班複製
- 班級名單匯入（格式：座號,姓名）
- 語音辨識核心：辨識「N號M分」格式填入表格
- 支援阿拉伯數字與中文數字（一號九十五）
- 統計分析：平均、90↑／80–89／70–79／60–69／60↓人數與分布圖
- 單班 CSV 匯出
- Safari / iPad 相容設計（continuous:false + 自動重啟）

#### v1.1｜語音辨識強化
- 新增 localStorage 自動儲存，重新整理後資料不遺失
- 修正中文數字解析：新增「兩字裸數字」支援（六一→61、七八→78）
- 新增「最近填入」標籤，點擊可直接修正
- 新增 Safari 諧音詞替換表（爸爸→八八=88、酒吧→六八=68 等）
- 修正單字數字判讀：「八」在號後且無「分」字→88，有「分」字→8

#### v1.2｜照片辨識匯入
- 新增照片辨識匯入功能（Claude JSON 貼入方案）
- 操作流程：複製提示詞 → 連同照片發給 Claude → 貼回 JSON → 解析預覽 → 確認匯入
- 提示詞自動產生（含座號範圍、JSON 格式範例）
- 新項目自動建立，不需預先設定
- 多科目一次匯入（如國文、英語、數學同時填入）
- 預覽表格可手動修改或刪除錯誤筆數後再匯入

#### v1.3｜重大 Bug 修正
- 修正 HTML 結構：panel-settings 缺少結尾 `</div>`，導致其他三個 tab 內容全部空白
- 修正 Safari `event.currentTarget` 問題：Safari 不支援全域 event 物件，改為 `switchTab(name, this)` 傳入元素
- 移除殘留的 `updateApiKeyStatus()` 呼叫（函式已刪除但 init 仍呼叫），導致頁面初始化中斷
- 移除重複的 `selectGradeClass` 函式定義，消除 JS 衝突

#### v1.4｜語音解析精化
- 修正「4號100 5號100」被辨識成「4號150號100」的問題
  - 原因：Safari 在停頓後重新辨識整段，將兩筆黏合並產生錯誤數字
  - 解法：`onresult` 只取 `event.resultIndex` 之後的新片段，不重新解析舊內容
- 分數正則限制為有效範圍（0–100），防止 150 被誤當分數
- 加分題處理：分數超過 100 自動截為 100

#### v1.5｜匯出功能升級
- 修正匯出 CSV 包含空白座位（34–40 號）的問題，改為只匯出有姓名或有成績的座號
- 全班匯出改為單一 `.xlsx` 檔案，各班各一個工作表分頁
- 使用 SheetJS（動態載入），Safari 相容

### 語音輸入使用注意

- 每說完一筆（例如「5號85」），**稍作停頓**並確認畫面已填入，再繼續念下一筆
- Safari 語音辨識會在停頓後重新辨識整段，系統已針對此行為最佳化，只解析新增片段
- 若辨識有誤，可點擊「最近填入」標籤直接修正，無需捲回表格

### 照片辨識使用注意

- 照片辨識需透過 Claude（此對話視窗）輔助，步驟：
  1. 在工具中複製提示詞
  2. 連同成績表格照片一起發給 Claude
  3. 將 Claude 回覆的 JSON 貼入工具
  4. 確認預覽後匯入
- 加分題若超過 100 分，Claude 應輸出 100；即使輸出超過 100，系統匯入時也會自動截為 100

### 已知限制

- 語音辨識依賴 Safari 內建語音辨識服務，需連線，辨識速度與準確率受環境影響
- 極少數諧音詞可能仍無法正確替換（如不在替換表中的誤辨情況），建議核對「最近填入」欄位
- Excel 匯出使用 SheetJS CDN，首次使用需網路載入
